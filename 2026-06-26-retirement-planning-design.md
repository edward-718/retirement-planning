# 养老准备系统设计文档

## 1. 项目概述

### 1.1 产品定位
一款面向全年龄段用户的**AI养老规划助手**，以"游戏化进度追踪"为前端交互核心，以"AI智能决策引擎"为后端骨架，帮助用户从年轻时就开始系统化地准备养老。

### 1.2 核心差异化卖点
1. **游戏化进度追踪**：将"存够200万养老"拆解为可视化的年度/季度里程碑，完成任务推进进度条，解锁成就，让长期储蓄变得有成就感
2. **养老地点智能推荐**：基于生活成本、医疗水平、气候舒适度、社区配套等维度，数据化对比全国养老城市
3. **AI养老顾问**：输入基本情况后，系统自动生成动态调整的长周期养老方案，每年自动复盘

### 1.3 目标用户
- **30-40岁**：需要长期规划教育和习惯养成
- **40-50岁**：需要精确测算和冲刺计划
- **50岁以上**：需要即时决策支持（地点选择、缺口确认）

### 1.4 竞争分析

| 产品/平台 | 核心功能 | 我们的差异化 |
|---|---|---|
| 中信银行"幸福+"养老账本 | 第一/二支柱测算、全资产展示 | 偏向银行账户整合，缺乏第三支柱规划和进度感 |
| 退休养老助手App | 养老金精算、财务健康评分 | 功能泛化，缺少游戏化推进和地点选择 |
| Boldin (海外) | 250+变量压力测试、模块化建模 | 中国本地化不足，缺少社保体系适配 |
| 美团养老 | 养老机构交易撮合 | 服务"即时刚需"用户，不做长期规划 |
| **本产品** | AI方案 + 游戏化进度 + 地点推荐 | **唯一覆盖"规划-追踪-决策"全链路**的工具 |

## 3. 核心功能设计

### 3.1 分层输入系统

| 模式 | 输入项 | 耗时 | 精度 | 适用场景 |
|---|---|---|---|---|
| **极简模式** | 年龄、当前城市、月收入、现有存款、目标养老城市 | 30秒 | 粗略估算 | 第一次体验、好奇测算 |
| **标准模式** | 极简项 + 社保缴纳年限、预期退休年龄、养老生活方式 | 2分钟 | 较准确 | 认真规划 |
| **专业模式** | 标准项 + 企业年金、个人养老金、投资明细、配偶信息、健康预期 | 5分钟 | 精准 | 临近退休、家庭联合规划 |

**AI兜底机制**：极简模式下，缺失数据由AI基于城市平均值和用户年龄推断，并在结果页提示"基于估算值，完善信息可获得更精准结果"。

### 3.2 AI决策引擎

**步骤一：三支柱养老金测算**
- 第一支柱：基于用户当前城市社保政策、缴纳年限、历年缴费基数，测算退休后每月可领取金额
- 第二支柱：用户手动录入企业年金数据（未来若与银行合作可自动同步）
- 第三支柱：跟踪个人养老金账户余额及投资收益

**步骤二：养老缺口计算**
```
总缺口 = 目标养老城市月生活费 x 预期养老年限 x 12
       - 第一支柱月领取 x 预期养老年限 x 12
       - 第二支柱月领取 x 预期养老年限 x 12
       - 第三支柱月领取 x 预期养老年限 x 12
       - 现有资产的未来终值（按预期投资回报率复利计算）
```

**步骤三：倒推储蓄计划**
- 根据总缺口、距退休年限、预期投资回报率，计算每月/每年需要新增储蓄额
- 若缺口为负，提示"可提前退休"或"提升生活品质"

**步骤四：动态调整触发条件**
- 每年自动复盘
- 用户收入发生重大变化
- 用户切换目标养老城市
- 后台更新社保政策模型参数

### 3.3 养老地点数据库

每个城市/社区包含以下维度：

| 维度 | 说明 |
|---|---|
| **生活成本** | 房租/房价、餐饮、交通、水电物业（按品质分三档） |
| **医疗资源** | 三甲医院数量、社区医院覆盖率、医保异地结算便利性 |
| **气候舒适度** | 年均温度、空气质量、湿度 |
| **社交/配套** | 公园密度、老年大学、社区活动中心 |
| **交通可达性** | 机场/高铁便利性 |

AI基于用户偏好权重（如"医疗 > 成本 > 气候"）进行排序推荐。

### 3.4 游戏化进度系统

| 元素 | 规则 |
|---|---|
| **总目标** | 用户自定义或AI建议（如"存够200万"或"实现月被动收入1万"） |
| **里程碑** | 系统自动拆解为年度/季度小目标 |
| **任务类型** | 储蓄任务、知识任务（读文章/看视频）、探索任务（了解一个城市） |
| **进度可视化** | 首页大图：总进度百分比 + 距离下个里程碑还差多少 |
| **成就体系** | "存下第一桶金"、"探索10个养老城市"、"连续储蓄12个月"等 |

## 4. 用户旅程

```
Day 1: 激活旅程
  打开小程序 -> 3分钟快速问卷
  (年龄/当前城市/收入/现有存款/目标养老城市/预期退休年龄)
  AI生成「养老蓝图」——一页纸看清：当前三支柱状态、缺口、建议月存额
  系统拆解第一个年度里程碑（如：今年存够15万，进度 0/15）

Week 1-4: 习惯养成
  每周推送「本周养老任务」
  完成任务推进进度条，解锁小成就

Month 3-12: 深度探索
  浏览「养老地点」板块，对比多个城市
  与AI对话："如果我去成都养老，每月需要多少钱？"

Year 1+: 动态复盘
  每年自动生成「年度养老复盘」
  根据实际存款、市场变化、目标调整，重新计算缺口和计划
```

## 6. 变现策略

### 6.1 核心原则
以**广告变现为主、会员订阅为辅**，最大化降低运营负担，适合兼职运营。

### 6.2 分阶段变现路径

| 阶段 | 变现方式 | 说明 | 运营投入 |
|---|---|---|---|
| **第一阶段** | 微信流量主广告 | UV达500/1000后开通，插入Banner/插屏/激励视频 | 零运营，自动跑 |
| **第二阶段** | 聚合广告平台 + 会员去广告 | 接入芒果联盟/uni-ad多平台预算池；推出会员去除广告+解锁高级功能 | 低运营，偶尔优化广告位 |
| **第三阶段** | 轻量机构合作 | 单城养老机构CPS导流（可选，视精力而定） | 每月几小时商务沟通 |

### 6.3 收入预期参考

| 日活用户 | 广告场景 | 预估月收入 |
|---|---|---|
| 1,000 | 首页Banner + 结果页插屏 | 300-800元 |
| 5,000 | Banner + 插屏 + 激励视频 | 2,000-5,000元 |
| 10,000 | 多广告位 + 聚合平台 | 5,000-15,000元 |

> 养老类用户消费能力强，金融/保险/健康类广告主出价高，实际eCPM可能高于普通工具类。

### 6.4 广告合规红线

- 禁止接"保本保息"、"稳赚不赔"等诱导性金融产品广告
- 禁止为无资质机构推广养老社区、保险、理财产品
- 优先接大健康、生活服务、电商、旅游类广告
- 所有广告内容需符合《金融产品网络营销管理办法》

## 7. 数据输入与对接策略

### 7.1 社保/年金数据现状

| 数据类型 | 开放API？ | 策略 |
|---|---|---|
| 第一支柱社保 | 不开放 | 用户手动输入 + AI智能默认值兜底 |
| 第二支柱企业年金 | 不开放 | 用户手动录入（架构预留对接模块） |
| 第三支柱个人养老金 | 仅金融机构间 | 用户手动录入（架构预留对接模块） |
| 城市生活成本 | 部分开放 | 爬取+人工校验公开数据 |

### 7.2 降低用户输入负担的设计

- 输入城市和年龄后，自动填充当地社保缴纳基数均值、默认已缴年限估算
- 不需要精确到每月，只需要"已缴多少年"、"月均缴费基数大约多少"
- 专业模式提供，但不强制

## 8. 风险与应对

| 风险 | 应对策略 |
|---|---|
| 金融合规风险 | 仅做"测算展示"，不做销售；广告严格审核；免责声明明确标注"测算结果仅供参考" |
| 用户信任建立慢 | 极简模式30秒出结果，先给价值再引导完善信息 |
| 数据准确性争议 | 标注数据来源和假设条件；提供"保守/中性/乐观"三种测算情景 |
| 竞品复制 | 核心壁垒在用户数据沉淀和AI个性化方案，非工具本身 |

## 9. 里程碑与成功指标

| 阶段 | 时间 | 目标 |
|---|---|---|
| **MVP** | 1-2个月 | 微信小程序上线：极简/标准测算 + 一页纸蓝图 + 基础进度条 |
| **V1.0** | 3-4个月 | 增加养老地点数据库 + AI问答 + 成就体系 + 微信流量主 |
| **V1.5** | 5-6个月 | Web端上线 + 家庭共享 + 压力测试 + 聚合广告 |

**核心成功指标**：
- 日活用户(DAU)
- 留存率（次日/7日/30日）
- 广告填充率和eCPM
- 会员转化率
