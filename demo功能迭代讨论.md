# 银龄智备 - Demo功能迭代方向讨论

## 一、Demo页面优先级：5选3策略

### 结论：优先实现这3个页面

| 优先级 | 页面 | 理由 |
|--------|------|------|
| P0 | **养老金测算器** | 核心工具页，是所有计算的起点，Demo必须展示"输入-计算"能力 |
| P0 | **养老蓝图** | 核心结果页，承接测算结果，用一页纸可视化呈现三支柱/缺口/建议，是用户价值最直观的体现 |
| P0 | **首页仪表盘** | 入口页+进度总览，承担导航和第一印象职责，同时可合并部分进度追踪元素 |

### 可简化/合并的页面

| 页面 | 处理方案 |
|------|----------|
| **进度追踪** | **合并至首页仪表盘**。Demo阶段无需独立页面，在首页以"我的养老进度"卡片形式呈现：进度条+2-3个待办任务即可 |
| **养老城市** | **保留但极度简化**。作为"Wow功能"存在，但Demo中仅展示6张城市卡片+基础对比，不做多维度筛选和智能算法包装 |

> **决策逻辑**：Demo的核心目标是"让评委/用户在3分钟内理解产品价值"。测算器（输入）+ 蓝图（输出）构成完整的最小闭环，首页提供入口和进度感知，三者足以讲述完整故事。

## 二、附件Demo功能与PRD页面对应关系

| 附件Demo功能 | 对应PRD页面 | 对应位置/模块 | 说明 |
|-------------|------------|--------------|------|
| **养老金缺口计算器** | 养老金测算器 | 核心表单区 | 附件功能即页面主体，极简模式（30秒版）可直接承载此计算逻辑 |
| **存款充足度追踪** | 首页仪表盘 | 核心数据卡片之一 | 以"养老资金准备进度"卡片形式嵌入首页，进度条+百分比+30%/55%/85%三档切换 |
| **养老地点智能推荐** | 养老城市 | 页面主体 | 6个城市卡片+生活成本vs舒适度二维指标，Demo中直接作为城市页全部内容 |

| **产品路线图** | 养老蓝图 | 蓝图页底部/侧边 | 4阶段规划作为蓝图页的"未来路径"模块，增强规划感 |

### 页面结构重组建议（Demo版）

```
首页仪表盘
├── 顶部：游戏化总览进度（存款充足度追踪）
├── 中部：快速入口（去测算 / 看蓝图 / 选城市）
└── 底部：养老准备行动清单（8项任务）

养老金测算器
├── 极简模式（默认）：年龄/收入/存款/替代率 4个输入
└── 计算结果 → 跳转养老蓝图

养老蓝图
├── 一页纸可视化：三支柱状态 + 缺口分析 + 储蓄建议
├── 产品路线图：4阶段规划
└── CTA：返回首页打卡任务 / 查看城市推荐

养老城市（简化版）
└── 6城市卡片网格：生活成本 vs 舒适度 散点/雷达图
```

## 三、游戏化元素在Demo中的实现程度

### 核心原则：**"视觉优先、逻辑从简"**

Demo阶段的游戏化不是为了构建完整系统，而是为了**传递"游戏化养老"的产品调性**。所有游戏化元素应在前端完成，不涉及后端状态持久化。

| 游戏化元素 | Demo实现程度 | 具体方案 |
|-----------|------------|---------|
| **进度条/百分比** | **完整实现** | 存款充足度追踪作为首页核心视觉，支持30%/55%/85%三档动画切换，数字滚动效果 |
| **行动清单勾选** | **完整实现** | 8项任务可点击勾选，勾选后即时反馈（打勾动画+轻微粒子效果），前端状态管理即可 |
| **里程碑** | **简化实现** | 在蓝图页展示4个阶段节点（如"起步期→积累期→冲刺期→享受期"），用解锁/锁定图标区分，Demo中预设1-2个已解锁 |
| **成就体系** | **视觉示意** | 首页展示2-3个成就徽章（如"初识养老""行动派"），Demo中可预设已获得，重点展示徽章的视觉设计而非完整解锁逻辑 |
| **积分/等级** | **暂不实现** | Demo阶段不做积分累计和等级系统，避免复杂度 |

### 关键决策

- **不做**：用户注册/登录、积分后端计算、成就解锁条件判定、社交排行榜
- **要做**：所有交互即时反馈、动画效果、视觉化的进度感知

## 四、Demo中的"Wow时刻"设计

为了让评委/用户在演示过程中眼前一亮，建议重点打磨以下4个时刻：

### Wow 1：一键生成的"养老蓝图"（核心Wow）
**触发**：用户在测算器输入4个数据后点击"生成我的养老蓝图"
**效果**：页面平滑过渡到蓝图页，数字以滚动动画呈现缺口金额，三支柱以动态图表填充，整个页面在2秒内"生长"出来
**价值**：直观感受"AI一键生成专属规划"的产品魔法

### Wow 2：存款充足度的"三档穿越"
**触发**：用户在首页点击30%→55%→85%切换
**效果**：进度条以弹性动画伸缩，数字滚动，同时背景色从橙（紧张）→黄（注意）→绿（充裕）渐变，配合表情图标变化
**价值**：让用户瞬间理解"我现在在哪，目标是哪"

### Wow 3：城市推荐的"散点图透视"
**触发**：进入养老城市页面
**效果**：6个城市不是平铺卡片，而是呈现在"生活成本-舒适度"二维坐标系中，用户一眼看出"高性价比区域"和"理想但昂贵区域"
**价值**：数据可视化的直观冲击力，体现"智能推荐"的科技感

### Wow 4：任务打卡的"成就绽放"
**触发**：用户勾选完成一项任务
**效果**：勾选瞬间有轻微粒子爆发效果，若该任务触发了某个成就，徽章以弹窗形式翻转出现
**价值**：游戏化的多巴胺反馈，让"枯燥的养老准备"变得有趣

## 五、Demo功能优先级清单（P0/P1/P2）

### P0 - Demo必须实现（阻塞级，没有则Demo无法闭环）

| 功能 | 页面 | 简化建议 |
|------|------|----------|
| 极简模式测算表单（年龄/收入/存款/替代率） | 养老金测算器 | 仅4个输入框+1个计算按钮，不做标准模式切换，不做复杂验证 |
| 测算结果计算逻辑（缺口金额/建议月存额） | 养老金测算器 | 使用简化公式前端计算，不做后端API，不做敏感数据加密 |
| 养老蓝图一页纸展示 | 养老蓝图 | 三支柱用3个色块进度条，缺口用大数字+红色强调，建议用2-3条bullet point |
| 首页仪表盘框架+导航 | 首页仪表盘 | 3个数据卡片（总进度/已打卡数/测算状态）+ 3个快速入口按钮 |
| 页面间路由跳转 | 全局 | 使用前端路由，页面切换加淡入淡出动画 |

### P1 - Demo强烈建议（极大提升演示效果）

| 功能 | 页面 | 简化建议 |
|------|------|----------|
| 存款充足度进度条+三档切换 | 首页仪表盘 | 纯CSS动画，3个预设按钮切换状态，不做真实数据联动 |
| 8项行动清单勾选打卡 | 首页仪表盘 | 前端状态数组管理，刷新后数据丢失可接受，勾选后即时视觉反馈 |
| 6城市卡片展示 | 养老城市 | 静态JSON数据，卡片展示生活成本/舒适度数值，不做真实数据库对接 |
| 4阶段产品路线图 | 养老蓝图 | 水平时间轴，4个节点，当前节点高亮，其余灰显 |
| 数字滚动动画 | 养老蓝图 | 金额数字从0滚动到目标值，增强数据呈现冲击力 |
| 蓝图页CTA按钮 | 养老蓝图 | "去打卡""看城市推荐"两个按钮，引导用户继续探索 |

### P2 - Demo锦上添花（时间充裕时再做）

| 功能 | 页面 | 简化建议 |
|------|------|----------|
| 成就徽章系统 | 首页仪表盘 | 预设2-3个已获得的徽章展示，不做解锁逻辑 |
| 城市二维散点图 | 养老城市 | 使用ECharts/Chart.js绘制，横轴生活成本纵轴舒适度，6个点+城市标签 |
| 测算结果分享卡片 | 养老蓝图 | 生成一张静态图片预览（可用html2canvas），展示"我的养老缺口是XXX" |
| 首页游戏化总览 | 首页仪表盘 | 一个综合完成度的环形图，整合任务/测算/存款三项进度 |
| 标准模式测算 | 养老金测算器 | 10+输入项的完整表单，Demo中可放但默认隐藏 |

## 六、总结：Demo最小闭环路径

```
用户打开首页 → 看到总进度+行动清单
       ↓
  点击"开始测算"
       ↓
  输入4项数据 → 点击计算
       ↓
  看到养老蓝图（Wow时刻：一页纸结果）
       ↓
  查看4阶段路线图 → 点击"去打卡"
       ↓
  回到首页勾选任务 → 进度条更新（Wow时刻：即时反馈）
       ↓
  点击"探索养老城市"
       ↓
  浏览6城市卡片（Wow时刻：散点图透视）
```

**核心指标**：评委/用户能在**3分钟**内走完这个闭环，理解"这是一款帮我做养老准备的游戏化AI工具"。
