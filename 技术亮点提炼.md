## 四、技术深度体现（可用于 Q&A 或技术文档）

### 4.1 精算引擎的技术细节

**文件**：`src/utils/calculator.ts`

- **社保养老金计算**：采用国内统账结合模型，基础养老金 = 退休时平均工资 × (1 + 缴费指数) / 2 × 缴费年限 × 1%，个人账户养老金 = 账户储存额 / 139
- **复利终值计算**：`FV = PV × (1 + r)^n`，含防御性检查（负利率/溢出/NaN 回退）
- **年金公式计算月储蓄**：`PMT = FV × r / ((1+r)^n - 1)`，处理零利率边界情况
- **三情景并行计算**：一次输入，同时输出保守(2%)/中性(4%)/乐观(6%)三种方案

```typescript
// 防御性编程示例
function calculateFutureValue(pv: number, rate: number, years: number): number {
  if (pv <= 0 || years < 0) return pv;
  if (rate < 0) return pv;
  const result = pv * Math.pow(1 + rate, years);
  return isFinite(result) ? result : pv * 100; // 溢出保护
}
```

### 4.2 动画系统的技术细节

**文件**：`src/components/ui/AnimatedNumber.tsx`、`src/index.css`

- **数字滚动动画**：基于 `requestAnimationFrame` + `easeOutCubic` 缓动函数，比 CSS transition 更精准控制时长和缓动曲线
- **级联入场动画**：通过 `animate-stagger-1` 到 `animate-stagger-6` 类名控制 0.1s~0.6s 的延迟，形成页面元素的波浪式入场
- **Confetti 粒子系统**：50 个随机位置/颜色/形状的 DOM 节点，使用 CSS `@keyframes confetti` 实现物理抛物线效果

```typescript
// easeOutCubic 缓动，让数字滚动更自然
const easeOut = 1 - Math.pow(1 - progress, 3);
```

### 4.3 可视化组件的技术细节

**文件**：`src/components/charts/CircularProgress.tsx`、`src/components/charts/FunnelChart.tsx`

- **SVG 环形进度条**：纯 SVG 实现，使用 `stroke-dasharray` + `stroke-dashoffset` 控制圆弧长度，支持渐变色描边
- **自定义漏斗图**：基于 CSS 过渡动画的声明式漏斗组件，自动计算各层宽度比例，支持负值（盈余）展示
- **图表联动交互**：Blueprint 页面切换情景时，PieChart/BarChart/AreaChart/FunnelChart 四图同步更新

### 4.4 表单校验的技术细节

**文件**：`src/pages/Calculator/index.tsx`

- **跨字段联合校验**：年龄字段校验时读取 `expectedRetirementAge`，退休年龄校验时读取 `age`
- **惰性校验策略**：仅在 `onBlur` 时标记字段为 `touched`，避免输入过程中频繁报错
- **动态字段渲染**：极简/标准模式通过同一个渲染引擎动态切换字段数量和步骤数

### 4.5 状态管理的技术细节

**文件**：`src/store/useStore.ts`

- **计算结果驱动进度**：`calculate()` 自动将计算出的总缺口映射到进度条目标值
- **成就系统联动**：`checkAchievements()` 在每次任务更新时自动判断 8 项成就的解锁条件
- **持久化配置**：使用 `zustand/middleware` 的 `persist`，指定 `name: 'retirement-planning-storage'` 作为 localStorage key

### 5.5 技术亮点文案（适合技术分享/面试项目介绍）

> 这个项目的核心挑战在于**把复杂的金融计算模型搬到浏览器里跑**。我参考了国内社保养老金的统账结合计算公式，用 TypeScript 实现了基础养老金 + 个人账户 + 企业年金 + 个人养老金的三支柱模型，全部在浏览器端用复利公式和年金公式实时计算。为了保证用户体验，我加入了防御性编程——处理负利率、数值溢出、NaN 等边界情况，确保任何输入都不会导致页面崩溃。
>
> 另一个亮点是**状态架构设计**。我用 Zustand 管理全局状态，通过 persist 中间件自动同步 LocalStorage。计算结果不仅是展示，还会驱动进度条的目标值更新；任务进度的变化会自动触发成就系统的检查。这种"计算-进度-成就"的联动，让状态管理不再是简单的数据存储，而是支撑整个应用逻辑的核心枢纽。
>
> 可视化层面，我没有简单调用图表库，而是**基于 Recharts 做了二次封装和联动设计**。Blueprint 页面的四种图表（饼图、柱状图、漏斗图、面积图）共享同一套计算结果，切换情景时四图同步更新。Cities 页面的散点图用气泡大小表达第三维度，雷达图展示五维评分，都是为了让抽象数据更容易被用户理解。
