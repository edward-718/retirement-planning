## 四、UI/UX 迭代方向

### 4.1 现有设计 vs 附件创意设计的融合策略

**总体策略：以现有实现为骨架，吸收附件的"排版节奏感"和"视觉层次"，不推翻重建。**

| 维度 | 决策 | 说明 |
|------|------|------|
| **色彩** | 融合 | 保持现有深青+暖金，增强金色"点缀感"，避免大面积金色疲劳 |
| **字体** | 借鉴策略 | 现有思源字体保留中文阅读优势，但采用"衬线标题 + 无衬线正文 + 等宽数字"三级策略 |
| **卡片** | 融合 | 保持 `rounded-2xl`，hover 阴影降级为更精致的 `shadow-lg` |
| **Hero** | 融合 | 附件全屏背景图+渐变遮罩视觉冲击力强，建议采用抽象暖色调渐变纹理或低饱和度风景图 |
| **计算器** | 取舍 | demo 保持步骤式（认知负荷低），但借鉴"左边框强调色"用于结果页数据卡片 |
| **任务清单** | 取舍 | 现有进度条适合储蓄类任务，成就墙可引入"解锁动画"，任务完成增加划线+淡出 |

### 4.2 Demo 最应优先落地的 5 个设计细节

1. **首页信息层级压缩（P0）**
   - 欢迎语压缩为 1 行 pill 标签 + 1 行主标题
   - 环形进度与 4 张数据卡合并为同一行（进度占 2 列，数据卡占 3 列）
   - 三支柱概览在 14 寸下默认折叠或移入第二屏

2. **全页面统一 Fade-in 滚动动画（P0）**
   - 引入 `IntersectionObserver` 触发的 `animate-fade-in-up`
   - 关键模块（卡片、图表、AI 建议）进入视口时依次浮现

3. **数据卡片"数字优先"排版重构（P1）**
   - 结构：图标（左上）→ 数字（居中偏大）→ 标签（底部小字）
   - 数字 `text-3xl/md:text-4xl font-serif`，标签 `text-xs uppercase tracking-wider`

4. **进度条高光泽质感（P1）**
   - 环形进度增加 SVG `linearGradient` 亮色节点模拟金属光泽
   - 里程碑卡片左侧边框增加 `rounded-full` 更柔和

5. **城市卡片选中态优化（P2）**
   - 选中后边框变 `border-gold-400` + 左上角勾选图标
   - 对比按钮常显但降低透明度，减少操作路径

### 4.3 响应式重点：14 寸笔记本一屏显示

14 寸常见分辨率 1366×768 或 1920×1080（系统缩放后有效高度约 600-700px）。

| 页面 | 14 寸适配策略 |
|------|--------------|
| **Dashboard** | 欢迎区 `mb-2`；数据卡 `p-4`；环形进度 size 降至 180；快速入口 4 列→2 列 |
| **Calculator** | 已有 `max-w-3xl`，适配良好，步骤指示器缩小圆点 |
| **Blueprint** | 图表 `h-72` → `h-56`；三支柱与漏斗图 `lg` 下单列堆叠 |
| **Cities 详情** | `lg:grid-cols-2` → 单列；图片 `h-64` → `h-48`；雷达图 `h-72` → `h-56` |
| **通用** | 引入 `xl` 断点（1280px）；垂直间距用 `space-y-6` 替代 `space-y-8` |

### 4.4 动效规范建议

```css
/* 页面加载 */
animate-fade-in-up: 0.6s ease-out;

/* 卡片 hover */
transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
transform: translateY(-4px);

/* 进度条填充 */
transition: width 1s cubic-bezier(0.4, 0, 0.2, 1);

/* 数字滚动 */
duration: 1000ms; easing: ease-out cubic;
```

## 五、技术实现迭代方向

### 5.1 单页 HTML → React 迁移策略

**策略：结构映射 + 样式转译 + 状态提升**

| 附件区块 | React 映射 | 状态方式 |
|---------|-----------|---------|
| 全局导航 | `Navbar.tsx` | 已存在 |
| 仪表盘 | `Dashboard/index.tsx` | Zustand `useStore` |
| 计算器 | `Calculator/index.tsx` | Zustand + 测算引擎 |
| 蓝图 | `Blueprint/index.tsx` | Zustand 订阅结果 |
| 进度追踪 | `Progress/index.tsx` | Zustand 里程碑/任务/成就 |
| 城市 | `Cities/index.tsx` | Zustand 城市数据 |

### 5.2 ECharts → Recharts 迁移

**结论：无需引入 ECharts，Recharts 完全覆盖需求。**

| ECharts 类型 | Recharts 对应 | 项目状态 |
|-------------|--------------|---------|
| 饼图/环形图 | `<PieChart><Pie>` | ✅ 已实现（三支柱） |
| 柱状图 | `<BarChart><Bar>` | ✅ 已实现（缺口分析） |
| 雷达图 | `<RadarChart><Radar>` | ✅ 已实现（城市五维） |
| 折线图 | `<LineChart><Line>` | ✅ 已实现（储蓄曲线） |
| **散点图** | **`<ScatterChart><Scatter>`** | ⚠️ **需补充**（城市成本 vs 舒适度） |

### 5.3 状态管理与持久化

**Zustand + LocalStorage 已配置完成，强烈推荐保留：**
- 刷新后数据不丢失，评审可展示完整用户旅程
- 实现成本极低（`persist` 中间件已就位）
- 纯前端 demo 的"持久化"唯一低成本方案

**Demo 阶段可精简：**
- 成就自动检查逻辑（约 50 行）
- 复杂情景模拟（保留一种即可）
- 任务 deadline 严格校验

### 5.4 技术债务修复清单

| 优先级 | 问题 | 位置 | 修复建议 |
|--------|------|------|---------|
| 🔴 高 | 城市图片 URL 依赖外部 AI API，不稳定 | `src/data/cities.ts` | 替换为 Unsplash 稳定链接或本地占位图 |
| 🔴 高 | `Dashboard` 中 `tips.content.replace` 返回值被误用为 JSX | `Dashboard/index.tsx:319` | 字符串替换改为 JSX 条件渲染 |
| 🟡 中 | 步骤配置与标准模式步骤数不匹配 | `Calculator/index.tsx:112-116` | `stepConfig` 扩展为 5 步匹配 `standardSteps` |
| 🟡 中 | 条件类名拼接冗长，无 `clsx` | 多处 | 安装 `clsx` + `tailwind-merge`，封装 `cn()` 工具函数 |
| 🟢 低 | 缺少城市成本 vs 舒适度散点图 | 无 | 新增 `ScatterChart` 组件 |
| 🟢 低 | 无滚动进入视口动画 | 无 | 自定义 `useInView` Hook 或 `framer-motion` |

### 5.5 建议补齐依赖

```bash
npm install clsx tailwind-merge
```

封装 `cn()` 工具函数：
```ts
import { clsx, type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';
export function cn(...inputs: ClassValue[]) { return twMerge(clsx(inputs)); }
```

## 七、迭代原则总结

1. **做减法**：Dashboard 首屏信息密度压缩，确保 14 寸一屏能看到核心数据
2. **做统一**：规范所有卡片的 hover、所有按钮的点击反馈、所有数字的字体层级
3. **做流动**：引入滚动触发的 fade-in 动画，让长页面有节奏感
4. **做精致**：阴影降级、金色克制使用、等宽数字对齐，提升金融工具的可信赖感
5. **保闭环**：测算器（输入）+ 蓝图（输出）+ 首页（进度）构成最小完整故事
